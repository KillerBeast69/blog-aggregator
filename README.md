## Overview

Gator is a command-line RSS feed aggregator that runs as a background service, fetching posts from registered feeds on a configurable interval and storing them in PostgreSQL. Multiple users can register independently, follow their own curated set of feeds, and browse a personalised, chronologically ordered post feed — all through a composable CLI. User session state is tracked in a lightweight local JSON config file at `~/.gatorconfig.json`, and all database interactions are generated as type-safe Go code by `sqlc`.

---

## Key Features

- 👤 **Multi-User Accounts** — Independent users persisted in PostgreSQL; the currently active user is tracked locally in `~/.gatorconfig.json` (stores DB URL + current username)
- 📡 **RSS 2.0 Feed Fetching** — Fetches feeds over HTTP using a custom `User-Agent: gator` header, then parses the XML response directly using `encoding/xml`; HTML entities in titles and descriptions are properly unescaped
- ⏱️ **Continuous Aggregation** — The `agg` command runs on a `time.Ticker` with a human-readable duration argument (e.g. `30s`, `5m`, `1h`), always selecting the least-recently-fetched feed first via `ORDER BY last_fetched_at ASC NULLS FIRST`
- 🗂️ **Feed Follow System** — Register and auto-follow new feeds with `addfeed`; follow or unfollow existing feeds by URL; list all global feeds or only followed ones
- 📰 **Personalised Post Browsing** — `browse [limit]` retrieves the N most recent posts across all followed feeds, ordered by `published_at DESC`; defaults to 2 if no limit is supplied
- 🛡️ **Login Middleware** — `middlewareLoggedIn` wraps all user-scoped commands, resolving the current user from the database before the handler runs and returning an error if no session is active
- 🗄️ **Type-Safe SQL** — All queries (users, feeds, follows, fetched timestamps, posts) are generated by `sqlc` from raw `.sql` files; no ORM

## Installation

**Prerequisites:** Go 1.22+, PostgreSQL, [`goose`](https://github.com/pressly/goose), [`sqlc`](https://sqlc.dev)

```bash
# 1. Clone the repository
git clone https://github.com/KillerBeast69/blog-aggregator.git
cd blog-aggregator

# 2. Install Go dependencies
go mod download

# 3. Create the local config file
cat > ~/.gatorconfig.json << 'EOF'
{
  "db_url": "postgres://username:password@localhost:5432/gator?sslmode=disable",
  "current_user_name": ""
}
EOF

# 4. Run database migrations
goose -dir sql/schema postgres "postgres://username:password@localhost:5432/gator?sslmode=disable" up

# 5. Build the binary
go build -o gator .
```

## Usage

```bash
# Register a new user (also sets them as the active session)
./gator register alice

# Log in as an existing user
./gator login alice

# List all registered users (* marks the active session)
./gator users

# Add a new RSS feed and automatically follow it
./gator addfeed "Wagslane Dev Blog" "https://www.wagslane.dev/index.xml"

# Follow an existing feed by URL
./gator follow "https://www.wagslane.dev/index.xml"

# List all feeds registered in the database
./gator feeds

# List feeds followed by the current user
./gator following

# Unfollow a feed by URL
./gator unfollow "https://www.wagslane.dev/index.xml"

# Start the aggregator — fetches all followed feeds every 1 minute
./gator agg 1m

# Browse the 10 most recent posts from your followed feeds
./gator browse 10

# Delete all users and their data (useful for development resets)
./gator reset
```

**Typical workflow:**

```bash
# 1. Start a session
./gator register alice

# 2. Add some feeds
./gator addfeed "Wagslane" "https://www.wagslane.dev/index.xml"
./gator addfeed "Boot.dev Blog" "https://blog.boot.dev/index.xml"

# 3. Run the aggregator in a separate terminal
./gator agg 30s

# 4. Browse posts as they arrive
./gator browse 5
```

## Database Schema

The five migrations create the following structure:

- **`users`** — `id (uuid PK)`, `name (text UNIQUE)`, `created_at`, `updated_at`
- **`feeds`** — `id (uuid PK)`, `name`, `url (UNIQUE)`, `user_id (FK → users)`, `last_fetched_at`
- **`feed_follows`** — `(user_id, feed_id) UNIQUE` — many-to-many join between users and feeds
- **`posts`** — `id (uuid PK)`, `title`, `url (UNIQUE)`, `description`, `published_at`, `feed_id (FK → feeds)`; `ON CONFLICT (url) DO NOTHING` prevents duplicate posts

## Contributing

Contributions are welcome. Fork the repository and open a pull request. New CLI commands should be registered in `main.go` using `cmds.register()`. Commands that require an authenticated session must be wrapped with `middlewareLoggedIn`. Database schema changes require a new numbered Goose migration file, a corresponding SQL query, and a `sqlc generate` run.
